## 📖 VAVE概述

### 1. 什么是VAVE？

**VAVE** = Value Analysis / Value Engineering

| 缩写 | 全称 | 含义 |
|------|------|------|
| VA | Value Analysis | 价值分析（已有产品） |
| VE | Value Engineering | 价值工程（新开发） |

**核心公式**:
```
价值(Value) = 功能(Function) / 成本(Cost)
       V = F / C
```

**目标**:
- 提高功能
- 降低成本
- 提升价值

### 2. 价值工程发展历程

| 阶段 | 时间 | 里程碑 |
|------|------|--------|
| 萌芽 | 1947 | 通用电器Lawrence Miles发明VE |
| 发展 | 1950s | 美国海军推广VE |
| 成熟 | 1960s | 成为国际标准 |
| 普及 | 1980s | 日本推广到家电、汽车 |
| 深化 | 2000s | 与TRIZ、DFMA结合 |

### 3. VAVE与传统成本降低的区别

| 对比项 | 传统降本 | VAVE |
|--------|----------|------|
| 目标 | 降低成本 | 提升价值 |
| 关注点 | 价格 | 功能/成本比 |
| 方法 | 压价、简化 | 功能分析、创新 |
| 结果 | 可能牺牲功能 | 功能不降低或提升 |

```

```

## 📝 第一步：功能分析

### 1. 功能定义

**定义**: 用简洁的语言描述功能

**规则**:
- 用动词+名词表达
- 避免使用"如何"、"怎样"
- 表达本质功能

**案例：电饭煲**
| 原描述 | 功能定义 |
|--------|----------|
| 把水加热成米饭 | 提供热能 |
| 把内胆放进去 | 支撑内胆 |
| 显示时间 | 指示时间 |

### 2. 功能分类

| 分类 | 定义 | 例子 |
|------|------|------|
| 基本功能 | 产品存在的根本理由 | 电饭煲煮饭 |
| 辅助功能 | 支持基本功能 | 内胆支撑 |
| 必要功能 | 用户需要的功能 | 煮粥、保温 |
| 不必要功能 | 用户不需要的功能 | 语音提示 |
| 过度功能 | 功能过剩 | 24种煮饭模式 |

### 3. 功能整理

**FAST图法** (Function Analysis System Technique)

```
              ┌─── 煮米饭 ───┐
              │              │
        ┌─────▼─────┐   ┌───▼─────┐
        │ 提供热能   │   │ 保温     │
        └─────┬─────┘   └─────┬─────┘
              │              │
        ┌─────▼─────┐   ┌───▼─────┐
        │ 控制温度   │   │ 显示状态 │
        └─────┬─────┘   └─────┬─────┘
              │              │
        ┌─────▼─────┐   ┌───▼─────┐
        │ 供电       │   │ 计时     │
        └───────────┘   └─────────┘
```

### 4. 功能评价

**功能重要性评分** (强制决定法 FD法)

| 零件 | 功能 | 重要性得分 |
|------|------|------------|
| 内胆 | 盛放米饭、传热 | 9 |
| 加热盘 | 提供热能 | 8 |
| 温控器 | 控制温度 | 7 |
| 外壳 | 保护内部 | 5 |
| 面板 | 用户交互 | 4 |
| 锅盖 | 密封保温 | 3 |

**功能现实成本**:
```
功能成本 = 零件成本分配

例如：内胆成本20元
- 盛放米饭功能分配：15元（75%）
- 传热功能分配：5元（25%）
```

## 🧮 第二步：成本拆解

### 1. 成本构成分析

**产品成本结构** (家电行业典型)

| 成本类型 | 占比 | 优化空间 |
|----------|------|----------|
| 直接材料 | 60-70% | 中 |
| 直接人工 | 5-10% | 小 |
| 制造费用 | 10-15% | 中 |
| 采购成本 | 70-80% | 大 |

### 2. 零件成本分解

**零件成本构成**:
```
零件成本 = 材料费 + 加工费 + 模具费分摊 + 管理费
```

**计算示例**:
| 零件 | 材料费 | 加工费 | 模具分摊 | 管理费 | 总成本 |
|------|--------|--------|----------|--------|--------|
| 内胆 | 8元 | 5元 | 2元 | 1元 | 16元 |
| 加热管 | 6元 | 3元 | 1元 | 1元 | 11元 |
| 外壳 | 10元 | 4元 | 3元 | 1元 | 18元 |
| PCB | 5元 | 2元 | 1元 | 1元 | 9元 |
| 其它 | 15元 | 8元 | 3元 | 2元 | 28元 |
| **合计** | **44元** | **22元** | **10元** | **6元** | **82元** |

### 3. 目标成本设定

**目标成本法**:
```
目标成本 = 售价 - 目标利润 - 税费
```

**示例**:
- 售价：299元
- 目标利润：30元（10%）
- 税费：27元（9%）
- **目标成本：242元**

**当前成本：282元**
**降本目标：40元（14%）**

## 💡 第三步：方案生成

### 1. 头脑风暴法

**规则**:
- 不批评任何想法
- 鼓励大胆设想
- 追求数量
- 互相启发

**提出问题**:
- 有没有其他方式实现同样功能？
- 能不能合并几个功能？
- 能不能用更简单的结构？
- 能不能用更便宜的材料？

### 2. 方案类型

| 类型 | 说明 | 效果 |
|------|------|------|
| 方案A | 功能不变，成本降低 | 基本目标 |
| 方案B | 成本不变，功能提升 | 增值方案 |
| 方案C | 功能提升，成本降低 | 创新方案 |
| 方案D | 略有降本，功能略降 | 妥协方案 |

### 3. 典型VE改进方向

| 方向 | 方法 | 案例 |
|------|------|------|
| 材料替代 | 用便宜材料 | 铜→铝、钢材→塑料 |
| 工艺替代 | 用便宜工艺 | 机加工→压铸 |
| 结构简化 | 减少零件 | 多个零件→1个 |
| 标准化 | 用标准件 | 定制件→标准件 |
| 模块化 | 通用化设计多种 | 型号共用模块 |
| 取消功能 | 删除不必要功能 | 删除极少用的功能 |
| 降低标准 | 合理降低 | 公差、精度 |

## 📊 第四步：方案评估

### 1. 功能评价法

**计算功能价值**:
```
功能价值(F.C.) = 功能现实成本 / 功能目标成本

判断标准:
- F.C. > 1：功能过剩，需改进
- F.C. = 1：理想状态
- F.C. < 1：需降低成本
```

**示例**:
| 功能 | 现实成本 | 目标成本 | F.C. | 建议 |
|------|----------|----------|------|------|
| 传热 | 15元 | 12元 | 1.25 | 优化结构 |
| 控温 | 10元 | 8元 | 1.25 | 简化设计 |
| 支撑 | 8元 | 10元 | 0.80 | 可接受 |
| 显示 | 6元 | 5元 | 1.20 | 降低标准 |

### 2. 成本评价法

**成本降低额**:
```
降本额 = 现实成本 - 目标成本

示例：内胆从20元降到15元，降本5元（25%）
```

### 3. 综合评价法

**加权评分法**:
| 方案 | 功能保持 | 成本降低 | 实施难度 | 总分 |
|------|----------|----------|----------|------|
| 方案A | 9 | 8 | 7 | 24 |
| 方案B | 8 | 9 | 6 | 23 |
| 方案C | 7 | 8 | 9 | 24 |

**评分标准**:
- 功能保持：1-10分
- 成本降低：1-10分
- 实施难度：1-10分（越低越好）

## 🛠️ VAVE实施工具

### 1. 成本模型

```python
# 成本计算示例
class CostModel:
    def __init__(self):
        self.parts = []
    
    def add_part(self, name, material, process, quantity):
        # 计算零件成本
        material_cost = self.calc_material(material, quantity)
        process_cost = self.calc_process(process, quantity)
        mold_cost = self.calc_mold(process, quantity)
        
        total = material_cost + process_cost + mold_cost
        self.parts.append({
            'name': name,
            'material': material_cost,
            'process': process_cost,
            'mold': mold_cost,
            'total': total
        })
    
    def get_total_cost(self):
        return sum(p['total'] for p in self.parts)
```

### 2. 价值评估

```python
def calculate_value(function_cost, target_cost):
    """计算功能价值"""
    return function_cost / target_cost

def assess_proposal(proposals):
    """评估VE提案"""
    for p in proposals:
        score = (
            p.function_score * 0.3 +
            p.cost_score * 0.4 +
            p.risk_score * 0.2 +
            p.implement_score * 0.1
        )
        p.final_score = score
    return sorted(proposals, key=lambda x: x.final_score, reverse=True)
```
