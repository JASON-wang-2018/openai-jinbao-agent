## 三、IT项目案例

### 案例1：跨国团队冲突管理

**场景**：美国、印度、中国成员开会冲突

**选项分析**：
- A. 组织团建 → 长期方案，不解决即时问题
- B. 引用文化差异 → 触及本质但非PMI首选
- **C. 依据团队章程** → ✅ 最佳实践，先依程序办事
- D. 上报发起人 → 过早升级，违背自主解决

**核心考点**：资源管理 - 冲突解决策略  
**PMP思维**：先依据既定规则

---

### 案例2：敏捷范围蔓延

**场景**：产品负责人冲刺中频繁添加需求

**解题要点**：
1. 识别问题：变更管理流程缺失
2. 排除错误：
   - ❌ 接受所有变更（违背基准控制）
   - ❌ 要求加班完成（忽视可持续节奏）
3. 正确路径：**召开变更控制委员会评估影响**

**数据**：2024年28% IT题涉及范围控制，62%与敏捷相关

## 四、建筑工程案例

### 案例1：材料变更连锁反应

**场景**：业主将玻璃幕墙改为石材幕墙

**影响分析**：
| 维度 | 具体表现 | 量化指标 |
|------|---------|---------|
| 时间 | 采购周期延长 | 关键路径+30天 |
| 成本 | 材料差价 | 预算超1000万 |
| 质量 | 安装工艺变化 | 验收标准修订12项 |
| 风险 | 结构承重变化 | 需重新安全评估 |

**正确步骤**：
1. 正式提交变更请求
2. 召集技术团队评估影响
3. 更新项目管理计划
4. 获取变更批准

**易错**：直接采纳变更未走流程

---

### 案例2：安全与进度冲突

**场景**：施工队建议省略安全防护措施赶进度

**PMI立场**：
- 安全标准是**刚性约束**（不可妥协）
- 进度问题通过其他合法途径解决

**正确选项**："拒绝妥协并探讨赶工方案"

**数据**：建筑工程题40%涉及合规困境，安全环保是高频考点

## 五、通用解题框架

### 四步决策模型

```
1. 情境定位 → 明确知识领域
2. 问题定性 → 识别问题类型
3. 流程匹配 → 对应PMBOK标准流程
4. 选项筛选 → 排除违背PMI价值观的选项
```

## 八、选项排除法则

### 通用排除原则

1. ❌ 首先排除违背职业道德的选项
2. ❌ 排除"绕过流程"的捷径类选项
3. ❌ 排除过度依赖职权的强制手段
4. ✅ 剩余选项中选最符合流程的
