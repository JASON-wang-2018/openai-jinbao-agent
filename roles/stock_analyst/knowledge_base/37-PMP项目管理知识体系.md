> PMP（Project Management Professional）项目管理专业人士
> 
> 核心理念：系统化、规范化、标准化管理项目

## 一、PMP知识体系概览

### 1.1 五大过程组

| 过程组 | 含义 | 关键活动 |
|--------|------|---------|
| **启动过程组** | 项目开始 | 制定项目章程、识别干系人 |
| **规划过程组** | 项目计划 | 制定项目管理计划、范围、进度、成本、质量、资源、沟通、风险、采购、干系人管理计划 |
| **执行过程组** | 项目实施 | 指导与管理项目工作、质量保证、获取资源、建设团队、实施采购 |
| **监控过程组** | 项目控制 | 监控项目工作、整体变更控制、范围验证、范围控制、进度控制、成本控制、质量控制、风险监控、采购控制 |
| **收尾过程组** | 项目结束 | 项目收尾、合同收尾 |

### 1.2 十大知识领域

| 知识领域 | 核心内容 |
|----------|----------|
| **整合管理** | 协调所有其他知识领域 |
| **范围管理** | 确保项目包含所需工作 |
| **进度管理** | 按时完成项目 |
| **成本管理** | 按预算完成项目 |
| **质量管理** | 满足干系人需求 |
| **资源管理** | 有效管理项目资源 |
| **沟通管理** | 确保项目信息及时准确 |
| **风险管理** | 识别和应对风险 |
| **采购管理** | 管理项目采购 |
| **干系人管理** | 争取干系人支持 |

```
项目
├── 子项目1
│   ├── 工作包1.1
```

### 4.2 范围蔓延 vs 范围镀金

| 概念 | 定义 | 处理 |
|------|------|------|
| **范围蔓延** | 客户不断提出新要求 | 通过变更控制管理 |
| **范围镀金** | 主动增加额外工作 | 避免，做好范围定义 |

## 五、项目进度管理

### 5.1 关键路径法（CPM）

```
关键路径 = 项目中最长的路径
关键活动 = 关键路径上的活动

特点：
├─ 关键路径上的活动没有浮动时间
├─ 关键路径决定项目最短工期
├─ 可以通过压缩关键路径缩短工期
└─ 项目中可能存在多条关键路径
```

### 5.2 关键链法（CCM）

```
关键链 = 考虑资源约束的关键路径

核心思想：
├─ 保护关键链
├─ 输入缓冲（FB）
├─ 输出缓冲（PB）
└─ 消除学生综合症
```

### 5.3 进度压缩技术

| 技术 | 说明 |
|------|------|
| **赶工** | 增加资源，缩短工期，成本增加 |
| **快速跟进** | 并行执行，缩短工期，风险增加 |
| **减少范围** | 缩小范围，缩短工期 |
| **提升效率** | 改进方法，提高效率 |

## 六、项目成本管理

### 6.1 成本估算方法

| 方法 | 适用场景 | 精度 |
|------|---------|------|
| **类比估算** | 历史数据丰富 | 低 (±20%) |
| **参数估算** | 历史数据+变量 | 中 |
| **三点估算** | PERT | 中 |
| **自下而上** | 详细工作 | 高 (±5%) |

### 6.2 挣值管理（EVM）

```
基本概念：
├─ PV（计划价值）= 应该完成的工作
├─ EV（挣值）= 实际完成的工作
├─ AC（实际成本）= 实际花费
└─ BAC（完工预算）= 总预算

绩效指标：
├─ CV（成本偏差）= EV - AC
├─ SV（进度偏差）= EV - PV
├─ CPI（成本绩效指数）= EV / AC
├─ SPI（进度绩效指数）= EV / PV
└─ EAC（完工估算）= BAC / CPI
```

### 6.3 挣值分析判断

| 指标 | >1 | =1 | <1 |
|------|-----|-----|-----|
| **CPI** | 成本节约 | 符合预算 | 成本超支 |
| **SPI** | 进度提前 | 符合进度 | 进度落后 |

## 八、项目资源管理

### 8.1 团队建设阶段（Tuckman模型）

```
形成阶段(Forming) → 震荡阶段(Storming) → 规范阶段(Norming) → 
解冻阶段(Performing) → 解散阶段(Adjourning)

团队建设策略：
├─ 形成：明确目标、角色
├─ 震荡：解决冲突、建立信任
├─ 规范：建立共识、形成文化
├─ 执行：高效协作、自组织
└─ 解散：工作交接、经验总结
```

### 8.2 冲突解决技术

| 方法 | 适用场景 |
|------|---------|
| **合作/解决问题** | 重要问题，双方合作 |
| **强迫/命令** | 紧急情况，一方主导 |
| **妥协/调解** | 双方平等让步 |
| **撤退/回避** | 暂时搁置 |
| **合作/包容** | 积累关系 |

## 九、项目风险管理

### 9.1 风险分类

| 分类维度 | 类型 |
|----------|------|
| **按影响** | 正面风险、负面风险 |
| **按来源** | 技术风险、外部风险、组织风险、项目管理风险 |
| **按可预测性** | 已知风险、可预测风险、不可预测风险 |

### 9.2 风险应对策略

**负面风险（威胁）**：
| 策略 | 说明 |
|------|------|
| **规避** | 改变计划消除风险 |
| **转移** | 转移给第三方 |
| **减轻** | 降低概率或影响 |
| **接受** | 承认风险，准备应急 |

**正面风险（机会）**：
| 策略 | 说明 |
|------|------|
| **开拓** | 确保机会必然发生 |
| **提高** | 增加概率或影响 |
| **分享** | 与第三方分享 |
| **接受** | 利用机会 |

### 9.3 风险登记册内容

```
├─ 风险编号
├─ 风险描述
├─ 概率和影响
├─ 风险评级
├─ 应对策略
├─ 责任人
├─ 触发条件
└─ 应急计划
```

| 方式 | 适用场景 | 反馈 |
|------|---------|------|
| **交互式沟通** | 会议、对话 | 即时 |
| **推式沟通** | 邮件、新闻 | 需要时 |
| **拉式沟通** | 知识库、门户 | 自主获取 |

### 10.2 沟通计算

```
沟通渠道数 = N(N-1)/2

示例：
├─ 3人团队：3条渠道
├─ 5人团队：10条渠道
└─ 10人团队：45条渠道
```

## 十一、干系人管理

### 11.1 干系人识别

```
干系人分类：
├─ 内部干系人： sponsor、团队成员、PMO
├─ 外部干系人： 客户、供应商、监管机构
└─ 权力/利益方格：
    ├─ 权力高/利益高：重点管理
    ├─ 权力高/利益低：令其满意
    ├─ 权力低/利益高：随时告知
    └─ 权力低/利益低：监督
```

### 11.2 干系人管理策略

| 策略 | 适用 |
|------|------|
| **管理** | 权力高、利益低 |
| **密切管理** | 权力高、利益高 |
| **随时告知** | 权力低、利益高 |
| **监督** | 权力低、利益低 |
