# 中高层管理者能力模型

> 更新时间：2026-03-04

---

## 一、核心能力框架

### 1.1 领导力三层模型

| 层级 | 核心能力 | 关键行为 |
|------|----------|----------|
| **基层** | 任务执行 | 帶團隊完成任務、培養執行力 |
| **中層** | 团队协调 | 跨部門溝通資源、向上向下管理 |
| **高層** | 战略决策 | 制定方向、培養接班人、塑造文化 |

### 1.2 管理者的五种角色（明茨伯格）

| 角色 | 定义 | 日常表现 |
|------|------|----------|
| 人际角色 | 领导人、联络者 | 面试、社交 |
| 信息角色 | 监控者、传播者 | 读报告、开会 |
| 决策角色 | 创业者、谈判者 | 做決定、資源分配 |

## 二、向上管理

### 2.1 核心原则

1. **结果导向** - 用结果说话，不找借口
2. **预期管理** - 及时同步进度，预警风险
3. **价值对齐** - 理解老板的核心诉求

### 2.2 汇报技巧

| 时机 | 内容 | 技巧 |
|------|------|------|
| 日常 | 进度/问题 | 结果先行+困难预警 |
| 专题 | 方案/成果 | 结论先行+数据支撑 |
| 紧急 | 突发状况 | 第一句话给结论 |

### 2.3 向上管理四要素

- **了解期望**：明确老板的核心KPI和关注点
- **主动同步**：超过2天的任务要定期汇报
- **提供选项**：问题带方案，不只抛问题
- **建立信任**：靠谱 = 说到做到 + 及时同步

## 三、向下管理

### 3.1 团队激励模型

**马斯洛需求层次在职场中的应用：**

| 需求层级 | 激励方式 |
|----------|----------|
| 生理需求 | 薪酬福利、工作环境 |
| 安全需求 | 稳定雇佣、社保公积金 |
| 社交需求 | 团队氛围、团建活动 |
| 尊重需求 | 公开表彰、授权赋能 |
| 自我实现 | 成长机会、挑战性任务 |

### 3.2 绩效面谈技巧

**BIC模型：**
- **B**ehavior（行为）- 具体发生了什么
- **I**mpact（影响）- 产生了什么后果
- **C**onsequence（后果）- 对团队/业务的影响

**正面反馈：及时 + 具体 + 公开**

### 3.3 辅导下属的GROW模型

| 阶段 | 任务 | 关键问题 |
|------|------|----------|
| **G**oal | 设定目标 | 想要达成什么？ |
| **R**eality | 了解现状 | 目前的障碍是什么？ |
| **O**ptions | 探索方案 | 有哪些选择？ |
| **W**ill | 确认行动 | 下一步怎么做？ |

## 四、跨部门协作

### 4.1 跨部门沟通原则

1. **利益共同体** - 找到共同目标
2. **资源交换** - 互惠互利
3. **流程梳理** - 明确接口人
4. **预设冲突** - 提前预警

### 4.2 推动跨部门项目的技巧

| 技巧 | 说明 |
|------|------|
| 共同KPI | 绑定各方利益 |
| 项目章程 | 明确职责边界 |
| 定期对齐会 | 保持信息同步 |
| escalation机制 | 快速升级问题 |

## 五、会议管理

### 5.1 高效会议原则

- **会前**：议题明确 + 资料提前发出
- **会中**：主持人控场 + 避免发散
- **会后**：纪要发出 + 任务落实到人

### 5.2 一对一（1-on-1）会议

**频率**：每两周至少一次，每次30分钟

**结构**：
1. 员工主导（70%时间）
2. 反馈交流（20%时间）
3. 方向对齐（10%时间）

**问题清单**：
- 最近有什么好消息？
- 遇到什么困难需要帮助？
- 有什么想尝试的新方向？

## 六、决策与问题解决

### 6.1 决策框架

**ORIDO模型：**
- **O**utcome - 期望结果是什么？
- **R**isk - 风险点有哪些？
- **I**mplementation - 如何落地？
- **D**ependency - 依赖什么？
- **O**utput - 交付物是什么？

### 6.2 问题分析工具

**5Why分析法：**
```
问题：项目延期
→ 为什么延期？需求变更频繁
→ 为什么需求变更？需求确认不充分
→ 为什么确认不充分？评审环节缺失
→ ...
直到找到根本原因
```

**鱼骨图（因果分析）：**
- 人：能力不足/态度问题
- 机：工具/系统
- 料：资源/物料
- 法：流程/制度
- 环：环境/文化

## 七、人才梯队建设

### 7.1 人才培养体系

| 层级 | 培养重点 | 方式 |
|------|----------|------|
| 高潜 | 战略思维 | 轮岗/项目历练 |

## 八、时间管理与效能

### 8.1 管理者时间分配

| 事项 | 建议比例 |
|------|----------|
| 团队管理 | 40% |
| 业务/专业 | 30% |
| 跨部门/对外 | 20% |
| 个人学习 | 10% |

### 8.2 优先矩阵

| | 紧急 | 不紧急 |
|---|------|--------|
| **重要** | 马上做 | 计划做 |
| **不重要** | 授权做 | 不做 |

### 8.3 日/周工作节奏

**每日**：
- 晨间：处理紧急事务
- 上午：核心工作时间
- 下午：会议/协作
- 临下班：复盘+次日计划

**每周**：
- 周一：目标对齐
- 周中：进度跟进
- 周五：复盘+团队1-on-1

## 九、职场政治与生存

### 9.1 职场生存法则

1. **不站队但要有盟友** - 保持中立，建立信任
2. **信息优先** - 信息即权力
3. **输出确定性** - 靠谱是第一位的
4. **懂得适时表现** - 酒香也怕巷子深

### 9.2 常见坑

| 坑 | 后果 | 避坑 |
|----|------|------|
| 只埋头干活 | 晋升慢 | 定期汇报+曝光 |
| 跨级汇报 | 被认为不忠 | 先告知直属 |
| 公开顶撞老板 | 信任崩塌 | 私下沟通 |
| 轻易承诺 | 失信 | 留有余地 |
