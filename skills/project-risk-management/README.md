## 🚀 快速开始

### 1. 创建风险登记册
```python
from risk_manager import RiskManager

rm = RiskManager()
df = rm.create_risk_register('风险登记册.xlsx')
```

这会创建一个包含10个典型风险的登记册，并用颜色标注风险等级：
- 🔴 红色：高风险
- 🟡 黄色：中风险
- 🟢 绿色：低风险

### 2. 添加新风险
```python
new_risk = {
    '风险描述': '服务器可能宕机，导致系统不可用',
    '风险类别': '技术',
    '可能性(1-3)': 2,  # 1=低, 2=中, 3=高
    '影响程度(1-3)': 3,
    '应对策略': '缓解',
    '应对措施': '做负载均衡，监控告警，灾备方案',
    '责任人': '技术总监'
}

rm.add_risk('风险登记册.xlsx', new_risk)
```

### 3. 更新风险状态
```python
# 风险已触发
rm.update_risk_status('风险登记册.xlsx', 'R001', '已触发',
                     trigger_event='周三服务器宕机2小时')

# 风险已缓解
rm.update_risk_status('风险登记册.xlsx', 'R002', '已缓解')

# 风险已关闭
rm.update_risk_status('风险登记册.xlsx', 'R003', '已关闭')
```

### 4. 生成风险报告
```python
report = rm.generate_risk_report('风险登记册.xlsx', '风险报告.md')
print(report)
```

报告包含：
- 风险概览（总数、等级分布、状态分布）
- 按类别分布
- 高风险列表（重点关注）
- 已触发风险
- 监控中风险
- 建议

## 📊 风险评估方法

### 可能性评估（1-3分）

**1分（低）**:
- 发生概率 < 20%
- 从未见过类似情况
- 可控因素多

**2分（中）**:
- 发生概率 20% - 60%
- 偶尔见到类似情况
- 部分可控

**3分（高）**:
- 发生概率 > 60%
- 经常见到类似情况
- 难以控制

### 影响程度评估（1-3分）

**1分（低）**:
- 影响范围小
- 影响程度轻
- 容易恢复
- 成本增加 < 10%

**2分（中）**:
- 影响范围中等
- 影响程度中等
- 需要一定时间恢复
- 成本增加 10% - 30%

**3分（高）**:
- 影响范围大
- 影响程度严重
- 很难恢复
- 成本增加 > 30%
- 可能导致项目失败

### 风险等级计算

```
风险等级 = 可能性 × 影响程度

1-3分：低风险 🟢
4-6分：中风险 🟡
7-9分：高风险 🔴
```

## 🛡️ 风险应对策略

### 规避（Avoid）
适用情况：高风险 + 高影响
措施：
- 放弃有风险的功能
- 选择成熟技术
- 增加人员备份
- 缩小项目范围

### 缓解（Mitigate）
适用情况：中高风险
措施：
- 增加测试
- 提前培训
- 预留时间
- 制定预案

### 转移（Transfer）
适用情况：中风险
措施：
- 外包
- 购买保险
- 签订合同
- 使用第三方服务

### 接受（Accept）
适用情况：低风险
措施：
- 监控跟踪
- 制定应急计划
- 预留预算

## ❓ 常见问题

**Q: 如何确定风险的可能性？**
A: 参考历史数据、团队能力、外部因素，结合判断

**Q: 风险等级如何计算？**
A: 风险等级 = 可能性 × 影响程度，1-3分低，4-6分中，7-9分高

**Q: 多久检查一次风险？**
A: 高风险每周，中风险每两周，低风险每月

**Q: 风险触发后怎么办？**
A: 立即执行应对措施，记录触发事件，更新风险状态

**Q: 如何降低风险？**
A: 提前识别（越早越好）、持续跟踪（不能松懈）、及时应对（快速响应）

## 🎯 最佳实践

### ✅ 要做的

1. **全员参与**
   - 每个团队成员都有责任
   - 鼓励上报风险
   - 不惩罚识别风险的人

2. **文档化**
   - 所有风险记录在案
   - 应对措施明确可执行
   - 更新历史可追溯

3. **定期回顾**
   - 每个里程碑都要回顾
   - 项目结束后复盘
   - 更新风险知识库

4. **持续改进**
   - 从每次风险中学习
   - 优化风险管理流程
   - 分享最佳实践

### ❌ 不要做的

1. **忽视低风险**
   - 低风险也可能升级
   - 持续监控很重要
   - 条件变化时重新评估

2. **只记录不行动**
   - 识别风险只是第一步
   - 要制定应对措施
   - 跟踪执行情况

3. **风险登记册一成不变**
   - 风险会变化
   - 新风险会出现
   - 定期更新

4. **忽视干系人**
   - 干系人期望也是风险
   - 定期沟通对齐
   - 管理期望
