### 2. First Principles Decomposer（第一性原理分解）
**位置**: `skills/first-principles-decomposer/`

**触发词**：
- "firstp", "first principles", "from scratch"
- "what are we assuming", "break this down"
- "atomic", "fundamental truth", "physics thinking"

**三阶段流程**：
1. **识别假设** - "我假设为真但可能不对的是什么？"
2. **分解到原子** - 不断问"为什么"直到触及基本事实
3. **从真理重建** - 只从已验证的基本事实构建解决方案

**输出格式**：
```
PROBLEM: [问题]

ASSUMPTIONS IDENTIFIED:
1. [假设] → Challenge: [为什么可能错]

FUNDAMENTAL TRUTHS:
• [基本事实1]
• [基本事实2]

REBUILT SOLUTION:
[从基本事实重建的方案]

VS CONVENTIONAL:
[与传统方法的不同]
```

### 3. Reasoning Personas（推理人设）
**位置**: `skills/reasoning-personas/`

**四种人设**：

#### Gonzo Truth-Seeker（真理探索者）
- **何时用**：探索想法、头脑风暴、打破局部最优
- **焦点**：找到缺口、挑战假设、不舒服的真相
- **问题**：哪里错了？缺少什么？大家在假设什么？

#### Devil's Advocate（魔鬼代言人）
- **何时用**：审查计划、做决策前、代码审查
- **焦点**：找到弱点、失败模式、风险
- **问题**：这怎么失败？最弱环节是什么？10倍规模会怎样？

#### Pattern Hunter（模式猎手）
- **何时用**：决策点、架构选择、"选A还是B"
- **焦点**：连接、先例、模式识别
- **问题**：什么相似？之前决定过吗？上次学到了什么？

#### Integrator（整合者）
- **何时用**：构建现有系统、确保连贯性
- **焦点**：系统连贯性、连接、整体视角
- **问题**：这怎么连接？影响什么？二阶效应？

**多人设分析**（最全面）：
1. Pattern Hunter - 上下文和先例
2. Gonzo Truth-Seeker - 新颖见解
3. Devil's Advocate - 失败模式
4. Integrator - 系统连贯性

## 技能整合应用

### 完整工作流

当用户遇到复杂问题时：

1. **First Principles** - 拆解问题到基本事实
2. **Pattern Hunter** - 寻找相似问题和先例
3. **Gonzo Truth-Seeker** - 找到新颖见解和缺失
4. **Devil's Advocate** - 找到失败模式和风险
5. **Integrator** - 确保方案与整体系统连贯
6. **Proactive Agent** - 提供用户没想到的建议

### 股票分析中的应用

#### First Principles（零和博弈）
```
假设：长期持有能赚钱
→ 挑战：市场是零和博弈，财富转移
→ 基本事实：散户整体注定亏（主力赚+手续费）
→ 重建方案：跟随主力，不看价格看量
```

#### Pattern Hunter（历史模式）
```
相似情况：之前春节后行情
→ 先例：科技股、消费股活跃
→ 学习：抓住持续性核心主线
```

#### Gonzo Truth-Seeker（挑战常识）
```
常见观点：价值投资、长期持有
→ 不舒服的真相：这是零和博弈，散户会被收割
→ 缺失：量价关系才是真实信号
```

#### Devil's Advocate（风险识别）
```
方案：跟进人形机器人
→ 失败模式：消息炒作后回落
→ 风险：高位追涨
→ 应对：等确认信号（放量突破）
```
