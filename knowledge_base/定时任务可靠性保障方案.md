# 定时任务可靠性保障方案

## 问题背景
定时任务可能因模型不支持、Gateway重启、网络问题等导致失败或错过。

---

## 一、排查问题

### 1.1 检查失败原因
```bash
# 查看所有任务的模型配置
cat /home/jason/.openclaw/cron/jobs.json | python3 -c "
import json,sys; d=json.load(sys.stdin)
for j in d.get('jobs',[]):
    model = j.get('payload',{}).get('model','N/A')
    print(f'{j[\"name\"]}: {model}')"
```

### 1.2 常见问题
- ❌ 模型不支持：`MiniMax-M2.1-lightning` → 改为 `MiniMax-M2.1`
- ❌ Gateway未重启
- ❌ 脚本错误

## 三、自动补偿机制

### 3.1 系统级 Cron（不消耗token）
```bash
# 添加到系统cron，每5分钟执行
(crontab -l; echo "*/5 * * * * /path/to/run_compensate.sh") | crontab -
```

### 3.2 工作流程
```
┌─────────────────┐
│  Gateway Cron   │ ← 定时触发任务
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 执行成功?       │ ──是──→ ✅ 记录成功
└────────┬────────┘
         │否
         ▼
┌─────────────────┐
│ 自动补偿 Cron   │ ← 每5分钟检查
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 补偿执行        │ → 发送通知/重试
└─────────────────┘
```

### 3.3 日志位置
- 补偿日志：`/tmp/cron_compensate.log`
- 执行历史：`skills/cron-manager/config/history.json`

## 四、使用方法

### 4.1 查看任务状态
```bash
python3 skills/cron-manager/cron_manager.py status
python3 skills/cron-manager/monitor.py
```

### 4.2 手动补偿
```bash
python3 skills/cron-manager/cron_manager.py catchup <任务名>
```

### 4.3 查看日志
```bash
tail -f /tmp/cron_compensate.log
```

## 六、相关技能对比

### Task Workflow（任务依赖管理）
- **仓库**：https://github.com/Charpup/openclaw-task-workflow
- **定位**：多步骤项目的任务依赖管理 + 进度追踪
- **功能**：
  - DAG调度（拓扑排序）
  - 复杂度评分（1-10）
  - 跨会话持久化
  - 自动迁移

### Task Scheduler（任务添加/删除）
- **仓库**：bussgrowwithlucky-crypto/openclaw-skill-task-scheduler
- **定位**：系统级 cron 任务管理
- **功能**：
  - 添加/删除定时任务
  - 验证 cron 表达式
  - 任务持久化到 JSON

### Cron Manager（定时可靠性）
- **定位**：确保定时任务不遗漏
- **功能**：
  - 任务执行记录
  - 健康检查
  - 自动补偿
  - 失败告警

### 三者关系
| 技能 | 解决的问题 |
|------|-----------|
| Task Scheduler | 任务"加/删"（添加、删除 cron 任务） |
| Task Workflow | 任务"怎么做"（流程、依赖、排序） |
| Cron Manager | 任务"别忘记"（监控、补偿、告警） |

**推荐组合**：
- **简单定时任务**：Task Scheduler + Cron Manager
- **复杂项目流程**：Task Workflow + Cron Manager
